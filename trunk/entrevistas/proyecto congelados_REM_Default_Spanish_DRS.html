1 Introducción

El proyecto esta basado en una empresa llamada Congelados Carmona S.L. Fue fundada sobre el año 92. En sus inicios esta empresa estaba situada en la Ctra. Nacional IV, km.509, 41410 Carmona (Sevilla). Posteriormente se traslado a una nave industrial en el Polígono El Pilero C/Cerrajero 2 también situado en Carmona.
Esta nave está dotada de una oficina, una zona para la venta al público, una cámara de congelación y otra de refrigeración con acceso privado para los trabajadores.
Congelados Carmona es una empresa encargada de ofrecer una gran variedad de productos, la mayoría del departamento de congelados y especialidades para restaurante, así como también se encarga de su servicio al público. Cuenta con una serie de proveedores que visitan el establecimiento y ofrecen productos nuevos, realizan compras a estos y ellos proponen la forma de pago. Hay distintas formas de pago a estos proveedores, al contado, a reposición, y con otros acuerdos (al mes, a 3 meses,?), estas formas de pago depende de la cantidad del pedido y del propio proveedor.

A lo largo de este documento vamos a describir los procesos relativos a la preventa y a la venta que realizan los trabajadores, así como la distribución del producto. Los trabajadores disponen cada uno de una cartera de clientes y estos los visitan, normalmente por las mañanas, por las tardes cargan las mercancías y preparan las facturas, al día siguiente la mercancía es distribuida y las formas de pago, al igual que los proveedores, dependen del cliente. Normalmente si el cliente lleva tiempo confiando en sus servicios, la empresa ofrece más facilidades de pago, y también dependiendo de la cantidad de la mercancía se acuerda una forma de pago u otra.
Esta empresa para garantizar un servicio optimo y para que los pedidos lleguen a tiempo y en buenas condiciones dispone de 4 furgones con cámara isotérmica. Por otra parte, otro servicio que presta esta empresa es la venta al público. Hay una zona de autoservicio en la que cualquier persona puede comprar y será atendida por una cajera que le cobrará al contado. El establecimiento está abierto de lunes a viernes de 10:00 a 20:00 de la tarde. La plantilla de trabajadores consta de dos repartidores, cinco vendedores, una dependienta y un administrativo.

Dependencias
  • [UC-0001] Dar de alta a un cliente
    • Descripción Crear un nuevo objeto de la clase Cliente en el sistema
    Parámetros DNI : String -- es el identificador del cliente
     NombreP : String -- nombre del cliente
     ApellidosP : String -- apellidos del cliente
     DomicilioP : String -- domicilio del cliente
     CPP : String -- codigo postal de la direccion del cliente
     LocalidadP : String -- localidad del cliente
     ProvinciaP : String -- provincia del cliente
     FechaDeNacimientoP : String -- fecha de nacimiento del cliente
     TelefonoP : String -- telefono del cliente
     PaisP : String -- pais del cliente
     emailP : String -- email del cliente
    Expresiones de precondición 1.: No existe un objeto de la clase Cliente en el sistema tal que su identificador DNI se corresponda con el DNI pasado como parámetro.
    Expresiones de precondición (OCL) 1.: true
    Expresiones de postcondición 1.: Se ha creado un objeto de la clase Cliente al que se le asigna como identificador DNI.
     2. : 2. Se asigna a los atributos nombre, apellidos, domicilio, CP, localidad, provincia, fechaDeNacimiento, telefono, país y email los valores nombreP,apellidosP, domicilioP, CPP, localidadP, provinciaP, fechaDeNacimientoP, telefonoP, paisP emailP pasados como parámetros al objeto desde la clase Cliente DNI = DNI.
    Expresiones de postcondición (OCL) 1.: true
    2. : true
